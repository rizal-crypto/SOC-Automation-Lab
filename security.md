# Security Policy

## 🛡️ Purpose of this Repository
This repository is a **Security Operations Center (SOC) Lab** designed for educational, research, and portfolio purposes. The configurations and scripts provided are intended to demonstrate defensive monitoring and incident response capabilities in a controlled environment.

## ⚠️ Ethical Use & Disclaimer
* **Educational Use Only:** The attack scripts (e.g., SMB Brute Force) included in this repository are for testing the detection capabilities of the SIEM. 
* **Authorized Testing:** These tools should only be used on systems you own or have explicit permission to test. 
* **No Warranty:** The configurations provided are for lab use and may not meet the hardening standards required for a production environment.

## 🛡️ Reporting a Vulnerability
I am committed to maintaining a secure and professional lab environment. If you discover a security vulnerability or a misconfiguration within this project (e.g., exposed sensitive data or unintended high-risk settings), please report it responsibly:

1.  **Do not** open a public GitHub Issue for a security vulnerability.
2.  **Contact me** directly via email at [rizalkapat8@gmail.com].
3.  I will acknowledge your report and work to remediate the issue as quickly as possible.

## 🛑 Safe Harbor
I support the "Safe Harbor" principle for security researchers. If you follow the guidelines above and act in good faith while investigating a potential issue in this lab, I will not pursue any legal action against you.
